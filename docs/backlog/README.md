# Backlog

This backlog turns the roadmap into pickable work items.

Use it to choose the next item for `$brainstorming` or `$planning`.

## How These Files Differ

```text
docs/milestones.md
  = the thin whole roadmap

docs/backlog/
  = pickable work items from the roadmap

.everything-automate/plans/
  = approved execution plans

.everything-automate/state/
  = execution progress and handoff state
```

Do not put detailed implementation plans in backlog items. A backlog item should be clear enough to pick, then `$planning` should make the detailed plan.

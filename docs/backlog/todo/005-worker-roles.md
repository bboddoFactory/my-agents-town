# Worker Roles

## State

todo

## Problem

Different models and agent services have different strengths.

The system needs a way to use more than one worker role without losing a common artifact flow.

## Desired Outcome

At least two worker roles can read an input artifact and write result artifacts in a comparable shape.

## Acceptance Criteria

- [ ] At least two worker roles can read an input artifact.
- [ ] Each worker role can write a result artifact.
- [ ] The orchestrator can explain why a worker was chosen.
- [ ] The user can compare role outputs.

## Notes

Do not build a broad agent marketplace.

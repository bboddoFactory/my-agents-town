# File Edit Review

## State

todo

## Problem

Worker edits should be reviewable before the user accepts them.

Stdout alone is not enough for file-changing work.

## Desired Outcome

A worker can make a small edit, and the UI can show changed files and a diff before accept or reject.

## Acceptance Criteria

- [ ] A worker can edit a controlled workspace.
- [ ] The system records changed files.
- [ ] The UI can show a diff.
- [ ] The user can accept or reject the result.
